# Build Roadmap

This project is built in six phases, each one a working, demoable product on its own.

- [x] **Phase 1 — Canvas + sync.** Multiplayer drawing/shapes/move, live cursors, presence. Single server instance, in-memory, no accounts yet. Shapes sync field-by-field (nested `Y.Map` per shape), so concurrent edits to different fields of the same shape both survive instead of one clobbering the other — verified with 3 simulated concurrent clients via the `multiplayer-sim-tester` subagent. Selection + delete (Delete/Backspace) and window-resize handling included.
- [x] **Phase 2 — Auth + friends + boards.** Login via Better Auth (email/password now, Google/GitHub wired but disabled until real OAuth credentials are added), a dashboard of your boards, adding friends by email, and inviting a friend to a specific board as collaborator or viewer. PostgreSQL via Docker from this point on. The centralized `authorization/boardAccess.ts` role check now gates both the REST API and the Yjs sync connection (via a short-lived signed "sync ticket") — verified with real HTTP requests and live Hocuspocus connections that a viewer's write is actually discarded server-side, not just hidden in the UI.
- [x] **Phase 3 — Chat.** DMs and group chats between friends, with typing indicators, scroll pagination, unread indicators, friendship-gated sending (disabled once unfriended, history preserved), per-user clear-history and delete-message, and WebSocket reconnect handling — verified end-to-end in the browser. Note to resolve when Phase 5 is designed: a board's live/broadcast chat needs to reconcile with this — likely a board-scoped chat is just a group chat whose membership mirrors `BoardMember`, not a separate system.
- [x] **Phase 4 — Video calls.** Camera/mic while sketching together, via a self-hosted LiveKit server. Role-gated at the token layer (`canPublish` derived server-side from board role, never client-controlled) — a viewer's client auto-connects subscribe-only whenever a collaborator/owner is actually in a call and auto-disconnects the moment none remain, with no manual Join Call control of their own. Floating, draggable, resizable call strip (client-side-only position/size, never synced). Verified end-to-end in the browser across owner/collaborator/viewer roles, including muted-state propagation and losing/regaining board access mid-call. Also closed a pre-existing Phase 1 gap found during this phase's testing: board canvas content is now persisted to Postgres (`Board.content`) instead of living in the Hocuspocus in-memory Y.Doc only.
- [ ] **Phase 5 — Anonymous viewer links.** A shareable, read-only link to watch a board's canvas, chat, and call — no account required. Chat mirrors real Twitch behavior: a **logged-in** viewer can post in chat (just not touch the canvas or publish media), while a fully **anonymous** link viewer is read-only on everything, chat included.
- [ ] **Phase 6 — Scale-out.** Add Redis (shared sessions + Hocuspocus's Redis extension), containerize everything with Docker Compose, run 2+ Node instances behind an nginx load balancer, and demonstrate two clients on different instances staying in sync.
